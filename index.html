Projects

Mobile Chat Application

In this project, I worked as the Scrum Master with a group with 3 other students to build a multi-functional chat mobile application for Android that integrates with a PostgreSQL Database and a Web Service hosted on Heroku. On the client side, I used Java in the Android Studio development environment. On the server side, I used javascript with node.js to write the application's backend, and used SQL with the integrated database.

The application includes the following functionality:

  • Secure sign-in with jwt storing for the ability for user to remain logged in
  • Secure Registration process with email verification
  • Setting that allows user to change their color scheme live
  • Backend API:
    • Handles secure sign-in and registration functionality
    • Upon registration of a new user, backend initiates an email verification process with NodeMailer.
    • Backend hooks into external weather API OpenWeatherMap to display weather data on the client-side.
    • Ability to detect device location to provide local weather information.
    • Uses Google Maps API to display a map widget to allow the user to select a location for weather data of that area.
    • Offers user ability to change their password, either within the app or as an external "forgot password" process.
    • Uses Pushy for instant notifications and visibility of new messages or contact requests.
    • Backend handles CRUD functionality for chat messages and contacts.
    • Ability to search for a person, send them a friend request, which can be approved or rejected by that user.
    • Offers user the ability to change their color scheme within the application.

Contact Tracing & Infectious Virus Tracking GUI & SQL Database

This project was created to allow medical professionals to perform easily perform CRUD operations to a database using a client application. The database was built to store patient data, medical provider information, medical visit records, potential transmission events, possible contaminees, test results, patient-reported symptoms, and prescriptions.

The application consists of the client side graphical user interface (GUI) written in Java, and a rationalized mySQL database containing 16 different tables. Parameterized database queries were utilized to defend against SQL injection.
